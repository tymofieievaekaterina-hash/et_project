# Определение переодов очередей на кассах
Расчет выполнен в MS Excel и MS SQL Server.
Как источник данных использована объединенная таблица из файлов "store_checkout_queues" и "store_stores". Объединение данных по чекам и магазинам, фильтрация данных по магазинам с id=98451680 и id=12864064 произведидино в Excel с помощью функционала надстройки Excel - Power Query. В результате получена таблицу вида:
checks_number
store_id
employees_id
quantity
selling_price
checkout_id
start_operation_dt
end_operation_dt

Такая же логика объединения и выборки магазинов воспроизведена в виде SQL запроса.

Логика расчета:
Операция считается находящейся в очереди, если время между текущей операцией и предыдущей менеше или ровно минимальному времени на выполнение одной операции конкретного кассира (учитывая количество позиций в чеке ) предидущей операции. Для расчета берем минимальное время на операцию кассира (выполняющего данную операцию) с таким же количеством товара как в предидущей операции, так как время выполнения операций зависит от количество позиций в ней.

Этапы расчета:
1. Агрегируем данные для расчета.
2. Определяем минимальное времени на операцию (в секундах) в зависимости от количества товаров в чеке для каждого кассира.
3. Определяем количество операцийв час.
